## Getting Started

### Prerequisites

- Node.js 18+ or Bun
- A Cosmic account with the Homiq bucket

### Installation

1. Clone the repository:
```bash
git clone <repository-url>
cd homiq-platform
```

2. Install dependencies:
```bash
bun install
```

3. Set up environment variables:
```bash
# .env.local
COSMIC_BUCKET_SLUG=your-bucket-slug
COSMIC_READ_KEY=your-read-key
COSMIC_WRITE_KEY=your-write-key
```

4. Run the development server:
```bash
bun run dev
```

5. Open [http://localhost:3000](http://localhost:3000) in your browser.

## Cosmic SDK Examples

### Fetching Properties with Location Relationships

```typescript
import { cosmic } from '@/lib/cosmic'

// Fetch properties with connected location data
const response = await cosmic.objects
  .find({ type: 'properties' })
  .props(['id', 'title', 'slug', 'metadata'])
  .depth(1) // Include connected location objects

const properties = response.objects as Property[]
```

### Fetching Trust Metrics (Singleton)

```typescript
// Fetch singleton trust metrics
const response = await cosmic.objects
  .findOne({ type: 'trust-metrics', slug: 'platform-statistics' })

const metrics = response.object as TrustMetric
```

### Filtering Properties by Location

```typescript
// Query properties by location ID (object relationship)
const response = await cosmic.objects
  .find({ 
    type: 'properties',
    'metadata.location': locationId // Query by ID, not nested properties
  })
  .depth(1)
```

## Cosmic CMS Integration

The application uses the following Cosmic object types:

- **Properties** - Property listings with verification status, pricing, features, and location relationships
- **Locations** - Cities with available home counts and average pricing
- **Features** - Verification steps and platform benefits
- **Testimonials** - Customer reviews with ratings and photos
- **Trust Metrics** - Platform statistics (singleton)

All content is managed through the Cosmic dashboard and fetched dynamically using the Cosmic SDK v1.5+.
